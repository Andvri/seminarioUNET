World Wide Web Consortium

  • Organización que controla los estándares web.
  • Creada y liderizada por Tim Berners Lee.
  • Intenta forzar compatibilidad entre navegadores.
  • Comité compuesto por representantes de varias empresas.
  • También provee educación sobre la web.

Etapas de la
especificación

  • Working Draft (WD): es un resumen que propone distintas características y funcionalidades. Generalmente propuesto por un representante para ser evaluado por el resto.
  • Candidate Recommendation (CR): cumple con todo lo que debería hacer según el autor.
  • Proposed Recommendation (PR): posee varias implementaciones que generan retroalimentación, se corrige hasta ser enviado a la W3C para su revisión.
  • W3C Recommendation: se recomienda a todos realizar la implementación.
A veces el estandar se maliterpreta y causa problemas, como el border entre IE y Netscape.

Conceptos erróneos
de HTML5

  • No es un lenguaje de programación.
  • HTML5 no incluye JavaScript ni CSS3.
  • HTML5 != Aplicaciones móviles.
  • HTML5 hace ver las aplicaciones parecer nativas.
  • HTML5 nunca será definido por completo.

HTML5

  • Lenguaje de marcado para la definición de información.
  • Introduce elementos semánticos para dar mayor significado a las distintas partes de un documento.
  • Sintaxis mas flexible: uso opcional de comillas, cierre de tags, entre otros.
  • Nuevas APIS: WebStorage, Drag and Drop, Canvas, audio y video nativo.
Algunas tecnologias como FileReader y WebAudio son especificaciones aparte.

CSS

Cascading Style Sheets

CSS

  • Creado en 1994 por Håkon-Wium-Lie (CTO de Opera Software).
  • Se utiliza para describir la apariencia y formato de un documento escrito en un lenguaje de marcado.
  • Permite separar el contenido de la presentación.
  • Estándar mantenido por la W3C.
La separación permite accesibilidad, compatibilidad con otros dispositivos, etc.

Preguntar sobre el lenguaje

Javascript

  • Creado por Brendan Eich para Netscape en 1995.
  • Inicialmente servía para ejecutar código del lado del cliente.
  • Ahora también se utiliza en el lado del servidor gracias a NodeJS.
  • Multiparadigma, sirve como: funcional, orientado a objetos, imperativo.
  • Especificado por ECMA-262
No es de la W3C, ES6, Microsoft intento hacer un fork
